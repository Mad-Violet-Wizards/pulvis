4. Core renderera (render)

To jest środek: komendy, kolejki, kamery, targety, warstwy.

4.1. Render target, viewport, kamera

class RenderTarget

owijka na GpuFramebufferHandle + rozdzielczość

int width, height;

struct Viewport

int x, y, w, h;

class Camera2D

Vec2 position;

float zoom;

float rotation; (jak chcesz)

Mat4 getViewMatrix() const;

Mat4 getProjMatrix(const RenderTarget&) const;

Mat4 getViewProj(const RenderTarget&) const;

(Później możesz dodać Camera3D, ale na razie 2D wystarczy.)

4.2. Materiał / parametry shaderów

Prosta wersja:

class Material

GpuProgramHandle program;

std::array<GpuTextureHandle, MAX_TEXTURE_SLOTS> textures;

mapka uniformów/parametrów (np. std::unordered_map<std::string, UniformValue>).

struct UniformValue

wariant: float, Vec2, Vec3, Vec4, Mat4, int, itp.

Jeśli chcesz bardziej low-level:

class UniformBuffer, class ShaderParamBlock – na później.

4.3. RenderCommand i RenderQueue

struct RenderCommand

referencja do DrawCall + high-level dane:

int sortKey;

int layer;

int passId;

class RenderQueue

void submit(const RenderCommand& cmd);

void sort();

void execute(GpuDevice& device);

Możesz trzymać w środku wektor DrawCall + dodatkowe meta.

4.4. Warstwy / kolejność

enum class RenderLayerId

Background, SpaceFar, SpaceNear, Planet, Foreground, UI, …

class RenderLayer

(opcjonalne) abstrakcja warstwy – tylko jeśli chcesz logicznie ogarniać typu: SpaceLayer, PlanetLayer.

RAW podejście:

po prostu w RenderCommand wpisujesz layer i sortujesz po nim.

4.5. Główny obiekt renderera

class Renderer2DCore (nazwa dowolna, np. Renderer)
Odpowiedzialny za:

trzymanie GpuDevice (albo referencję),

trzymanie RenderQueue,

API w stylu:

beginFrame(RenderTarget& target, const Color& clearColor);

submit(const RenderCommand& cmd);

endFrame(); // sort + execute

6.5. LOD dla planety

9. Co jest absolutnym minimum na start?

Jeśli chcesz zacząć małym krokiem, to MVP to:

gfx::GpuDevice + handle typu GpuTextureHandle, GpuBufferHandle, GpuProgramHandle, DrawCall.

render::Camera2D, RenderTarget, Renderer2DCore (+ RenderQueue).

render2d::SpriteBatch + prosty shader 2D.

render2d::TileSet, TileMap, TileChunk, TileChunkBuilder, TileMapLayer.

Reszta (PlanetRenderer, LOD, Starfield, Nebula, PostProcessChain) może dojść później, na tej samej architekturze – nic nie trzeba będzie wywalać.

Jeśli chcesz, w kolejnym kroku mogę wziąć np. moduł render2d i rozpisać nagłówki klas (.hpp) z dokładnymi polami/metodami tak, żebyś mógł je od razu wklejać do projektu i stopniowo implementować.
